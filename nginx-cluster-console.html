源配置
目标节点与写入目录
替换模式要求目标 Agent 0.3.0 或更高版本，并会记录被替换文件的原始 Hash。
!
每个目标节点都会先暂存配置并执行 nginx -t，通过后才原子写入和 reload；失败自动恢复。成功节点会加入此站点的托管范围，原节点配置不会删除。若配置引用证书，请先确保目标节点已有对应证书。

从节点移除配置

选择要删除配置文件的节点
只允许删除 Agent 托管配置目录中的 .conf 文件；离线、缺少当前 Hash 或 Agent 版本过低的节点不可选择。
!
Agent 会校验文件当前 Hash，暂存恢复副本后删除，再执行 nginx -t 和 reload；任一步失败都会恢复原文件。这里只移除所选节点上的配置文件，不删除平台站点记录，也不删除证书。

添加证书

上传证书与私钥，并写入所选节点的托管证书目录

私钥上传后不可查看、不可下载，也不会写入页面状态或执行日志。
替换目标
替换时保持原文件路径不变；Agent 会先校验证书与私钥，再原子覆盖并执行 nginx -t 和 reload，失败自动恢复旧文件。
